OVERVIEW FOR DEPLOYING WINDOWS UPDATES

The following procedures describe how to install the latest Windows OS versions. You can accomplish this by first creating a Group Policy Object (linked to an Organizational Unit), loading a WSUS administrative template, configuring automatic updates, and then specifying update service locations. Windows OS updates can be configured using Group Policy settings for Windows Server Update Services (WSUS). The Group Policy Object (GPO) will be linked to an Active Directory container appropriate for your environment. Configuring Windows OS updates also includes Internet Explorer browser updates.

NOTE: We do not recommend editing the Default Domain or Default Domain Controller GPOs to add WSUS settings.

CREATING A GROUP POLICY OBJECT (GPO)

  1. Start GPMC using one of the following methods:
    • Click Start > Administrative Tools > Group Policy Management.
    • Click Start, then type gpmc.msc in the text box and press Enter.
  2. In the GPMC console tree, right-click Group Policy Objects folder in the domain in which you want to create a GPO and selectNew.
  3. In the New GPO dialog box, enter a name for the New Group Policy Object (Windows OS Updates) and click OK. The new GPO appears in the Group Policy Objects folder.
  4. Right-click on the OU where you want to apply the policy and click Link an Existing GPO.
  5. Select the GPO name created in step 3 and click OK. The Group Policy is now linked to the selected OU.
